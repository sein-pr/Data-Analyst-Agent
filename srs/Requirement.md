# Project: Autonomous Self-Healing Data Analyst Agent

## 1. Objective
Build an agentic workflow that monitors a Google Drive folder for new Excel files, performs "self-healing" semantic data cleaning (handling column renames/additions), generates business insights using an LLM, and produces a professional PowerPoint report automatically.

## 2. Tech Stack
- **Orchestration:** Python (deployed via Modal)
- **Data Storage:** Google Drive (via Google Drive API)
- **LLM Engine:** Gemini (for reasoning and narrative)
- **Data Processing:** Pandas, Openpyxl
- **Reporting:** Python-pptx
- **Hosting:** Modal.com (for serverless execution and cron-triggers)

## 3. Core Workflow (The Agentic Loop)

### Phase 1: The Watcher (Trigger)
- **Input:** A specific Google Drive Folder ID.
- **Action:** The agent polls the folder every X hours (or via webhook).
- **Filtering:** Identify new `.xlsx` or `.csv` files that haven't been processed yet.
- **Lifecycle:** Once a file is picked up, move it to a `/processing` subfolder to avoid duplicate runs.

### Phase 2: Self-Healing & Semantic Mapping (The "Brain")
Traditional BI fails when columns change. This agent must:
1. **Extract Schema:** Read the raw headers of the new file.
2. **Semantic Alignment:** - Send the raw headers to the LLM.
   - **Prompt:** "I need columns for 'Revenue', 'Date', and 'Product Category'. The user provided: [List of raw headers]. Map the raw headers to my needs. If 'Drive_Rev' looks like 'Revenue', map it. Identify any NEW columns not in my core set."
3. **Data Normalization:** Use the mapping to rename the DataFrame columns dynamically before analysis.

### Phase 3: Insight Generation
- **Analysis:** Use Pandas to calculate Key Performance Indicators (KPIs) like MoM Growth, Top 5 Products, and Outliers.
- **Interpretation:** Feed the summarized results to the LLM.
- **Goal:** Generate 3-5 "Executive Bullet Points" per slide that explain the "Why" behind the numbers, not just the "What."

### Phase 4: PowerPoint Construction
- **Tool:** `python-pptx`.
- **Slides:**
  - Slide 1: Title & Executive Summary.
  - Slide 2: KPI Dashboard (using tables/charts generated by the agent).
  - Slide 3: Deep Dive into New/Unexpected Data (the "Self-Healing" report).
  - Slide 4: Recommendations.

## 4. Technical Constraints & Security
- **API Credentials:** Use `service_account.json` for Google Drive access. Store as "Secrets" in the hosting environment (Modal).
- **Token Efficiency:** Do not send entire raw datasets to the LLM. Only send column names for mapping and *aggregated summaries* for insight generation.
- **Error Handling:** If the agent cannot find a "Revenue-like" column, it must log an error and notify the user via a status file in Google Drive.

## 5. Implementation Roadmap for the AI Agent
1. **Step 1:** Setup Google Drive API connection and "file watcher" logic.
2. **Step 2:** Develop the `DataHealer` class that handles the semantic mapping via LLM.
3. **Step 3:** Create the `AnalysisEngine` to process dataframes.
4. **Step 4:** Build the `PPTXGenerator` to map insights to slide templates.
5. **Step 5:** Wrap the entire flow in a Modal function for deployment.
